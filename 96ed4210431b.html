    项目启动初期后端开发人员需要讨论需求、设计数据库、讨论需求合理性等工作。这期间会占用大量的开发时间，而这段时间前端开发人员则会相对比较清闲。

    如何在项目开发期间摆脱后端接口束缚又能模拟真实的接口请求？

    Mockjs

    生成随机数据，拦截 Ajax 请求。

    安装

    推荐使用yarn安装。

    快速开始

    在项目根目录 src 下新建 mock 文件夹并新建 index.js 文件。

    Mock语法

    Mock.mock( rurl?, rtype?, template|function( options ) )

    • rurl: 表示需要拦截的 URL，可以是 URL 字符串或 URL 正则
    • rtpe: 表示需要拦截的 Ajax 请求类型。例如 GET、POST、PUT、DELETE 等。
    • template: 表示数据模板，可以是对象或字符串。
    • function: 表示用于生成响应数据的函数。
      • options: 指向本次请求的 Ajax 选项集，含有 url、type 和 body 三个属性。

    数据模板template定义规范

    数据模板中的每个属性由属性名、生成规则、属性值构成。属性名和生成规则之间用竖线|分隔。

    例如：

    生成规则的7种格式

    1. name|min-max: value
    2. name|count: value
    3. name|min-max.dmin-dmax: value
    4. name|min-max.dcount: value
    5. name|count.dmin-dmax: value
    6. name|count.dcount: value
    7. name|+step: value

    Mockjs 是一种比较老的技术了，很多人仅仅只是听说过，实际中并没有使用过。但是当能够真正熟练定义数据模板时，它是可以为你的开发带来很大的便利的。
